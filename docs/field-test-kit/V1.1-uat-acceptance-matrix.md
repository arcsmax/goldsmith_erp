# V1.1 UAT Acceptance Matrix

**Source:** TESTABILITY-REVIEW.md §2 (Lena Vogel, UX / QA), amendment A13.3.
**Purpose:** the 8-step field-test script for Meister Thomas's workshop.
**Owner:** Max Kull.
**Cadence:** runs once during internal dogfood (W9, 2026-06-09 to 2026-06-15)
and once on ship-day (2026-06-16 Tue) as part of on-site training.

Each step has five columns:

| Pre-condition | Action | Expected state | Evidence | Pass criterion |

A step is PASSED when every column's condition is met by the recorded
evidence. A single FAIL in steps 1–5 occurring twice across the dogfood
team halts deployment per `V1.1-dogfood-protocol.md` blocker list.

User-facing copy in the Expected state / Action columns is in German to
match the product. Reviewer notes and headers stay in English.

## Step 1 — FAB scan starts a timer from cold

| Pre-condition | Action | Expected state | Evidence | Pass criterion |
|---|---|---|---|---|
| No running timer for the user. Phone camera permission granted (or freshly accepted). Printed label `ORDER:1` on matte stock ready. | Meister tippt auf FAB ("Scannen"), zeigt die Kamera auf das Label `ORDER:1`. | Exactly 1 row appears in `time_entries` with `origin='scan'`, `status='active'`, `user_id` = Meister's id. Pubsub `time_tracking_updates` envelope emitted with `source: "scan"`. The DB row exists and is the only one for this user in the last minute. |

## Step 2 — Scan a second order switches the timer atomically

| Pre-condition | Action | Expected state | Evidence | Pass criterion |
|---|---|---|---|---|
| Step 1 passed and the timer for `ORDER:1` is still active. Label `ORDER:2` ready. | Meister scannt `ORDER:2`. Modal "Quick Action" erscheint, erste Option ist "Timer wechseln". Tippt auf "Timer wechseln". | The `ORDER:1` entry is ended (`status='stopped'`, `end_time` set) AND a fresh entry for `ORDER:2` starts, in **one** DB transaction. No intermediate state where both timers are active or both are stopped. | Screen video + DB diff: rows for `ORDER:1` and `ORDER:2` have neighbouring timestamps; no row with two active timers for the same user. | Atomic switch; elapsed from scan-decode to TimerWidget reflecting ORDER:2 <= 5 s (spec §14.a row b p95 target). |

## Step 8b — Admin cannot bypass the Punzierungs guard

| Pre-condition | Action | Expected state | Evidence | Pass criterion |
|---|---|---|---|---|

Admin token. | Admin calls `PATCH /orders/99 {"status":"COMPLETED"}`. | Backend returns `HTTP 409` with code `PUNZIERUNG_NOT_VERIFIED` (Slice 5 A5.3).
