## Scenario matrix (8 scenarios)

A "completed dogfood day" for a participant = they hit ≥ 3 of these
during normal work. Every scenario is hit by at least one participant
each day.

| # | Scenario | Fallback / edge-case emphasis |
|---|---|---|
| 1 | FAB scan starts a timer from cold. | None. |
| 2 | Scan-to-switch-timer across two orders. | Same user, back-to-back orders. |
| 3 | `ACTIVITY:` scan mid-session updates the current entry. | No new row expected. |
| 4 | Deliberate alloy mismatch + override with category + reason. | B3 PII regex-deny triggered on long name input (expected). |
| 5 | Deliberate Punzierungs-Check block + resolve. | Modal blocks COMPLETED until ≥ 1 Feingehalt mark checked. |
| 6 | Camera denied → manual fallback path. | Autofocus numeric keypad, sub-3-second entry target. |
| 7 | USB HID session on shared tablet with Werkbank-Station-Modus ON. | Browser autofill / 1Password does NOT fire listener. |
| 8 | Network drop (airplane mode) during an action. | Observe failure mode; no stuck-spinner, no duplicate rows on reconnect. |

Track scenarios in `dogfood-log.csv` (columns below).

---

## Daily cadence

Daily, 15-minute standup at 09:00:

1. **Spot-check the CSV.** Who hit < 3 scenarios yesterday? Any FAIL
   rows?
2. **Top-3 bug triage.** Henrik + Lena pick. Blocker-severity bugs
   (security, data loss, stuck states) escalate to Henrik same-day;
   UX issues go to Lena with a repro video.
3. **Go / no-go signal for the day.** Any blocker condition triggered
   (see next section) → surface to Max immediately.

The rest is routine app use — no choreography.

## Deployment blockers (binary)

**Any single one of these unresolved at EOD 2026-06-15 Mon halts
ship.** These are not "watch items" — they are binary gates.

| # | Blocker condition | Source | How detected |
|---|---|---|---|
| B1 | Scenario 1–5 fails ≥ 2× across participants (not necessarily the same one twice, two independent failures in scenarios 1-5). | Lena §5 | CSV FAIL count per scenario. |
| B2 | `BenchScannerListener` fires with `activeElement` ≠ `<body>` even once during Scenario 7. | Lena §5 | scan_logs + DevTools `activeElement` snapshot. |
| B3 | Bundle-size CI gate flaps between green / red across PRs during the dogfood week. | Lena §5 | `.github/workflows/bundle-size-gate.yml` history. |
| B4 | FAB-tap-to-timer p95 > 8 s on the workshop tablet across any 24-hour window. | Lena §5 | `/admin/scan-gate` latency tile. |
| B5 | Any `user_id` field accepted in a request body by any V1.1 endpoint. | Lena §5 (security regression) | Integration test sweep / penetration test. |
| B6 | VIEWER sees a financial field in a `/scan/resolve` response. | Lena §5 | Playwright test on `allow-list` projection. |

If any blocker fires: halt, document on `V1.1-real-human-validation-
checklist.md`, slip to 2026-06-23 Tue (see `V1.1-ship-day-policy.md`).

## Logging format — `dogfood-log.csv`

Stored next to this file at `docs/field-test-kit/dogfood-log.csv`. One
row per scenario execution:

| Column         | Type / values                                          |
|----------------|--------------------------------------------------------|
| date           | `YYYY-MM-DD`                                           |
| user           | `henrik` / `anna` / `kai` / `backend_dev` / `lena`     |
| scenario_id    | `1` .. `8`                                             |
| pass           | `pass` / `fail` / `partial`                            |
| duration_s     | integer seconds from start-of-scenario to completion   |
| notes          | freetext, no PII (no customer names / contact info)    |
| screenshot_url | optional — path or sentry-attachment link              |
| blocker_flag   | `B1` / `B2` / `B3` / `B4` / `B5` / `B6` / empty        |

Each row with non-empty `blocker_flag` is a halt signal. Review daily.

An empty starter CSV ships in this directory so participants can
commit directly into it.

## What dogfood is NOT

- **Not a replacement for the UAT acceptance matrix.** That runs on
  ship-day with Meister Thomas, not internal users.
- **Not a security audit.** Henrik's separate penetration test / code
  review agent dispatch covers B5/B6 before dogfood starts.
- **Not a performance test at scale.** We only have 5 users, on real
  networks, with realistic usage patterns. Load testing is a separate
  V1.1.5 task if the adoption curve demands it.
