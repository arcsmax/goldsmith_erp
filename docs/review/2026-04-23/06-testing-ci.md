- **Modules with zero integration tests (routers):** `activities`, `admin_email`, `analytics`, `calendar`, `comments`, `customer_portal` (has `test_portal.py` — actually covered), `handoffs`, `health`, `imports`, `invoices`, `materials`, `measurements`, `metal_prices`, `metal_types`, `notifications`, `photos`, `scrap_gold`, `theme`, `time_tracking` (start/stop path has no dedicated endpoint test; only `test_switch_timer_endpoint.py` covers switch), `users`.
- **Coverage gate:** CI enforces `--cov-fail-under=50` (backend). No enforced coverage floor for frontend (Vitest coverage reporter is configured in `vitest.config.ts:12-22` but is never requested by CI).

## CI pipeline health

`ci.yml` is active, parallel, and reasonably structured: two lint jobs (backend `black/isort/bandit/mypy`, frontend `tsc --noEmit`) feed two test jobs, then a PR-only container build. Postgres 15 + Redis 7 are provisioned as services for the backend test job, and `alembic upgrade head` runs against them before pytest — that is effectively the migration smoke test (but only the forward path; no downgrade executed in CI).

Key weaknesses: **(1)** backend tests themselves use SQLite via `tests/conftest.py:33` and `tests/integration/conftest.py:42` regardless of the `DATABASE_URL=postgresql+asyncpg://...` env var the CI job sets — so the Postgres service is used *only* for `alembic upgrade`, not for the test suite. Tests never run against the production dialect. **(2)** E2E Playwright specs are committed but not in any CI workflow. **(3)** No Alembic downgrade verification. **(4)** `pylint` is declared as a dev dep and used in `make lint`, but is not called in CI — dead tooling. **(5)** `mypy --ignore-missing-imports` in CI effectively neuters `[tool.mypy] strict = true` from `pyproject.toml`. **(6)** `bundle-size-gate.yml` only runs when `frontend/**` changes; that's correct for that gate, but there is no generic frontend-test trigger that runs on backend-only changes, which is fine.

The pre-commit config (`.pre-commit-config.yaml`) covers gitleaks + bandit + black + isort + ruff + standard hygiene — all backend-focused; no frontend hook (eslint/prettier) runs on commit, and in fact the frontend has no eslint/prettier config at all.

## Critical test gaps (prioritized)

1. **Real-Postgres test run** (P0) — Switch at least the integration suite to run against PG in CI so the `test_concurrent_metal_consumption.py` FIFO/LIFO money-calc tests are not silently skipped, and so migrations + ORM models are exercised against the production dialect.
2. **Alembic upgrade→downgrade→upgrade smoke test** (P0) — Catch the class of migration bugs commit `263fd45` alludes to ("3 real bugs caught by smoke-test against fresh PG") inside CI, not after merge.
3. **Playwright E2E in CI** (P0) — Four specs are committed; 0 of them run. The highest-value flow (login → create order → start timer → stop timer) should be a single smoke E2E that blocks merge.
4. **Un-skip encryption tests** (P0) — `test_encryption_key_configured` and `test_encryption_decrypt` in `tests/test_basic_setup.py` are skipped with a stale reason. Encryption is a GDPR P0 control; the tests must actually execute.
5. **Permission-decorator enforcement matrix** (P1) — `test_auth_permissions.py:177-193` covers `require_permission` at the unit level, but there is no systematic test that iterates every `@require_permission`-decorated route and asserts it returns 403 for the wrong role. With 30+ routers this should be auto-generated, not hand-written.
6. **Time-tracking integration lifecycle** (P1) — `test_switch_timer_endpoint.py` only covers `/switch`. Add coverage for start/stop/manual/delete + interruption-log over the real router + WebSocket broadcast.
7. **WebSocket publish→delivery end-to-end** (P1) — Current WS tests only cover the auth handshake with `subscribe_and_forward` patched. Add a fakeredis-backed test asserting a published event is actually forwarded to the client socket.
8. **Notifications router** (P1) — Not covered anywhere. Security-sensitive (it surfaces erasure-request updates) and user-facing.
9. **Frontend page tests** (P1) — `OrdersPage`, `ScannerPage`, `TimeTrackingPage` need at least render-plus-MSW interaction tests. Currently 0% coverage at the page layer.
10. **Auth flow token-expiry & refresh** (P1) — `test_token_refresh.py` exists in unit/ but there is no integration test that proves an expired JWT is rejected with the correct error payload from a real request cycle.
11. **GDPR export shape** (P2) — Erasure is well-tested (`test_gdpr_customer_erasure.py`, 45 tests). Art. 15 export endpoints need an equivalent suite.
12. **Bandit / pip-audit in CI** (P2) — `bandit` runs; `pip-audit` does not. `yarn npm audit` doesn't run either. Dependency CVEs land silently.

## Strengths observed

- **Real behavioral tests, not mock theatre.** 990 test functions, 2112 assertions, 258 DB commits — tests actually exercise code paths, not just import modules. Mock usage is targeted (123 occurrences, mostly for Redis/external services).
- **Thoughtful fixture design.** `tests/integration/conftest.py` provides `admin_auth_headers`, `goldsmith_auth_headers`, `viewer_auth_headers`, and `authenticated_client` — the `authenticated_client` fixture you'd want to exist does exist and is used consistently.
- **Isolation hygiene.** Unique per-test email suffixes (`_unique('admin')`) and per-test table wipe prevent row-leak collisions; rate-limiter reset (`tests/conftest.py:107-119`) prevents cumulative `5/minute` login caps from flaking the suite.
- **Migration tests are substantial.** `test_migration_qr_core.py` (550 lines) and `test_migration_slice_2.py` (872 lines) both cover upgrade *and* downgrade paths, not just forward migration.
- **Pub/sub failure semantics tested.** `test_pubsub_failure_handling.py` (474 lines) explicitly verifies the `_safe_publish` contract across three services — exactly the kind of error-path test that is usually skipped.
- **Frontend MSW setup is correct.** `frontend/src/test/setup.ts` uses MSW with `onUnhandledRequest: 'error'` — unknown requests are caught, not silently stubbed. Vitest + `happy-dom` + `@testing-library` is the modern stack.
- **Playwright config is right-shaped** (`playwright.config.ts`): auto-starts Vite, retries twice in CI, screenshots on failure. Just not wired up.
- **Gitleaks + bandit + ruff + black + isort** on every commit (where not bypassed) is a strong pre-commit floor.
- **Bundle-size gate exists and runs on PR** (`bundle-size-gate.yml`) — a sophisticated addition most projects of this size do not have.
